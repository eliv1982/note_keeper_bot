# Как добавить Git в PATH (чтобы команды работали в терминале Cursor)

Терминал в Cursor использует переменную окружения **PATH** вашей системы (или пользователя). Если `git` не находится, его папку нужно добавить в PATH.

## Windows

### 1. Узнать, где установлен Git

Обычно Git ставится в одну из папок:

- `C:\Program Files\Git\cmd`
- `C:\Program Files (x86)\Git\cmd`

Проверьте, что там есть файл `git.exe`. Если ставили через winget или в другое место — найдите папку, в которой лежит `git.exe`.

### 2. Добавить папку в PATH

**Через «Параметры системы»:**

1. Нажмите Win + R, введите `sysdm.cpl`, Enter.
2. Вкладка **Дополнительно** → кнопка **Переменные среды**.
3. В блоке **Переменные среды пользователя** (или **Системные**) выберите переменную **Path** → **Изменить**.
4. **Создать** → вставьте путь к папке с `git.exe` (например `C:\Program Files\Git\cmd`) → **ОК** во всех окнах.

**Через «Параметры Windows» (Windows 10/11):**

1. Пуск → **Параметры** → **Система** → **О программе** → **Дополнительные параметры системы** (или поиск «переменные среды»).
2. **Переменные среды** → в **Path** пользователя или системы → **Изменить** → **Создать** → вставить путь к папке с Git → **ОК**.

### 3. Перезапустить Cursor

Новый PATH подхватывается только в новых процессах. Закройте Cursor полностью и откройте снова — после этого в терминале Cursor должна находиться команда `git`.

### 4. Проверка

В терминале Cursor (или PowerShell) выполните:

```powershell
git --version
```

Если выводится версия Git — PATH настроен.

---

**Примечание:** Если Git ставился через установщик с [git-scm.com](https://git-scm.com), при установке обычно предлагают «Add Git to PATH». Если тогда был выбран другой вариант, добавьте путь вручную, как выше.
